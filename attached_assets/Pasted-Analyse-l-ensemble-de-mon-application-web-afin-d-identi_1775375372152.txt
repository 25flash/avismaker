Analyse l’ensemble de mon application web afin d’identifier les points d’amélioration et proposer des optimisations globales.

Objectif :
Améliorer la performance, l’expérience utilisateur (UX), la qualité du code et la maintenabilité de l’application.

1. Audit global :
- Analyser la structure du projet (frontend + backend si présent)
- Identifier les problèmes d’organisation du code
- Détecter les composants dupliqués ou inutiles
- Vérifier la cohérence globale (UI, logique, nommage)

2. Performance :
- Identifier les lenteurs (chargement, rendu, requêtes)
- Optimiser le rendu des composants
- Réduire les appels inutiles (API, re-renders)
- Améliorer le lazy loading si applicable
- Optimiser les images et assets

3. UX / UI :
- Détecter les frictions utilisateur (navigation, compréhension)
- Améliorer la lisibilité et la hiérarchie visuelle
- Simplifier les parcours utilisateurs
- Vérifier la cohérence des composants (boutons, formulaires, cartes)
- Proposer des améliorations concrètes page par page

4. Responsive :
- Vérifier l’affichage sur mobile, tablette et desktop
- Corriger les problèmes de layout (overflow, tailles, espacements)
- Uniformiser le comportement responsive

5. Accessibilité :
- Vérifier les contrastes, tailles de texte
- Ajouter les attributs nécessaires (aria, labels)
- Améliorer la navigation clavier

6. Qualité du code :
- Nettoyer le code inutile ou redondant
- Améliorer la lisibilité et la structure
- Factoriser les composants réutilisables
- Appliquer les bonnes pratiques du framework utilisé

7. Sécurité (si backend) :
- Vérifier la gestion des données sensibles
- Sécuriser les endpoints API
- Valider les entrées utilisateur

8. Fonctionnalités :
- Identifier les fonctionnalités manquantes ou améliorables
- Proposer des améliorations à forte valeur (UX ou business)

Contraintes :
- Ne pas casser les fonctionnalités existantes
- Proposer des améliorations progressives
- Prioriser les optimisations avec le plus d’impact

Résultat attendu :
- Une liste claire des problèmes identifiés
- Des recommandations priorisées (high / medium / low impact)
- Des exemples de code corrigé si pertinent
- Des suggestions d’amélioration UX concrètes et applicables immédiatement

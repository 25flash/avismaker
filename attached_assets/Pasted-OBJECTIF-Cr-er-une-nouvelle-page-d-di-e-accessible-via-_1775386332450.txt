OBJECTIF :
Créer une nouvelle page dédiée accessible via /business-analytics avec :
- Analytics avancées
- Export PDF

IMPORTANT :
Les fonctionnalités doivent être FULL accessibles uniquement pour les utilisateurs Business,
mais visibles (floutées + bloquées) pour Free et Premium pour inciter à l’upgrade.

---

1. PAGE /business-analytics

Créer une page dashboard moderne avec :

HEADER :
- Titre : "Analytics avancées"
- Sous-titre : "Suivez vos performances en temps réel"
- Bouton : "Exporter en PDF"

SECTIONS :

1) KPI (cards)
- Total scans
- Taux de conversion
- Croissance (%)
- Activité récente

2) GRAPHIQUES
- Courbe d’évolution (scans dans le temps)
- Répartition des sources (Google, Airbnb, etc.)
- Top cartes

3) TABLEAU
- Nom carte
- Nombre de scans
- Source
- Performance

---

2. GESTION DES PLANS (CRITIQUE)

SI user.plan === "business" :
- Accès total
- Graphiques interactifs
- Export PDF actif

SI user.plan === "premium" OU "free" :

- Appliquer :
  filter: blur(6px)
  pointer-events: none

- Ajouter un overlay centré :
  🔒 Icône lock
  Texte : "Débloquez les Analytics avancées avec Business"
  Sous-texte : "Accédez aux statistiques complètes et exportez vos rapports"
  Bouton : "Passer à Business"

- Ajouter :
  - Animation fade-in
  - Hover sur bouton
  - Tooltip : "Disponible avec Business"

---

3. EXPORT PDF

- Bouton visible pour tous
- Actif uniquement si Business
- Sinon → ouvrir modal upgrade

PDF doit contenir :
- Logo
- Date
- KPI
- Graphiques
- Tableau résumé

Librairie suggérée :
- jsPDF ou html2canvas

---

4. NAVIGATION

Ajouter dans la sidebar :
- "Analytics avancées"
- Icône graphique
- Badge "Business"
- Icône cadenas si non Business

5. BACKEND (SECURITE)

Créer route :
GET /api/business-analytics

Protection :
if (user.plan !== "business") {
  return 403
}

- Context global ou middleware

  "Vous pourriez générer +32% avec ces insights"
